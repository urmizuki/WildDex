PROBLEM STATEMENT

THE COMPLEXITY

Accurate tree species identification in tropical forests requires highly specialized expertise in dendrology and wood anatomy.

THE SHORTAGE

There is a severe shortage of on-site experts available in the field.

THE IMPACT

This slows biodiversity auditing, delays forest research, and weakens enforcement of logging regulations.

2 / 12
OUR SOLUTION

An AI-powered "Point-and-Identify" mobile app for field foresters, researchers, and citizens.

REAL-TIME RESULTS

Instant species identification via smartphone camera — bark textures or leaf morphology.

METHODOLOGY
01

DATASET SOURCING

Compiled localized Malaysian timber datasets. Trained TensorFlow.js + Teachable Machine on 5 species: Jati, Meranti, Cengal, Keruing, Kulai. Confidence threshold 0.5.

02

APP ARCHITECTURE

Next.js 15 SSR layer for auth + Supabase. Vanilla JS SPA in iframe for scanning. Pixel-art CSS design system. Per-user localStorage state.

03

UX DESIGN

Onboarding intro → Camera scan (HUD overlay) → AI prediction → Card reveal → Collection exploration (herbarium, impact, journal).

04

FEEDBACK LOOP

Track every scan attempt. Expedition tiers (1-50 scans) incentivize repeated use. Data identifies model gaps for retraining.

5 / 12
TECHNICAL ARCHITECTURE

LAYER 1: NEXT.js SSR

Supabase Auth (email/password). App shell renders SPA in iframe. Password reset API via Supabase Admin.

LAYER 2: VANILLA SPA

8 pages (Home, Scan, Collection, Reveal, Profile, Impact, Journal, Subscription). TensorFlow.js inference in browser. localStorage state.

DATA FLOW

1. User captures image (camera/upload)
2. TF.js model predicts species
3. If confidence >= 0.5, show card reveal
4. Save to collection + journal entry
5. Update expedition tier
6. Re-render views

AI MODEL DETAILS

MODEL

Teachable Machine image classifier trained on bark textures and leaf morphology. 5 classes with 0.5 confidence threshold.

RUNTIME

Browser-side inference via TensorFlow.js. No server required. getUserMedia API for camera access.
